# 招投一体数字员工 (Investment & Promotion AI Agent) - 产品文档

## 1. 产品说明书 (Product Manual)

### 1.1 产品定位
“招投一体数字员工”是专为地方政府招商部门、国资投资平台及产业园区设计的智能办公助手。它通过 AI 技术深度融合“财务投资研判”与“产业招商逻辑”，实现从项目筛选、深度分析到会议管理的全流程数字化。

### 1.2 核心功能模块
*   **招投一体驾驶舱**：提供区域招商大局观，实时监控产业链补链进度、项目转化率及未来税收预测。
*   **项目处理中心**：
    *   **财务视角**：自动分析创始团队、市场规模、财务状况及风险点。
    *   **招商视角**：量化项目与本地产业的契合度，识别迁址信号，自动匹配扶持政策。
*   **会议纪要录音**：支持访谈录音一键转录，自动提取关键决策与待办事项，形成专业投资纪要。
*   **智能产业咨询**：基于本地政策库的 AI 问答，随时解答招商政策与产业规划疑问。

### 1.3 使用流程
1.  **项目导入**：在“项目处理中心”上传企业 BP。
2.  **AI 研判**：查看 AI 生成的双维分析报告，评估项目落地价值。
3.  **会议记录**：在项目考察或投决会时开启录音，自动生成纪要。
4.  **决策辅助**：参考驾驶舱宏观数据，结合项目微观分析，做出招投决策。

## 2. 需求规格说明书 (Requirements Specification)

### 2.1 功能需求 (Functional Requirements)
*   **FR-01: 多维数据可视化**：系统需支持展示项目总数、产业链完成率、税收预测等核心指标。
*   **FR-02: 自动化 BP 解析**：系统需能识别 PDF/文本格式的 BP，提取核心商业信息。
*   **FR-03: 产业契合度算法**：需建立模型评估项目对特定区域（如嘉兴南湖区）的补链贡献。
*   **FR-04: 语音转文本与摘要**：需集成语音识别与 NLP 技术，实现会议内容的结构化提取。
*   **FR-05: 历史存档管理**：需支持项目库与会议库的分类存储与多条目检索。

### 2.2 非功能需求 (Non-Functional Requirements)
*   **安全性**：所有项目数据需严格加密存储，符合国资数据安全规范。
*   **响应速度**：AI 分析响应时间需控制在 30 秒以内，页面切换需流畅无延迟。
*   **易用性**：界面需符合金融办公审美，操作逻辑简洁，降低用户学习成本。

## 3. UI 设计说明 (UI Design Documentation)

### 3.1 设计哲学
本系统采用 **“现代金融简约风” (Modern Financial Minimalist)**。
*   **色彩规范**：以“墨黑”与“纸白”为主基调，辅以“翡翠绿”（代表增长与成功）和“宝石蓝”（代表科技与严谨）。
*   **排版逻辑**：采用大留白设计，强调信息层级。使用 **Inter** 字体确保数字与文字的极佳阅读体验。
*   **组件化设计**：所有卡片采用 `rounded-3xl` 大圆角，降低视觉压迫感，提升亲和力。

### 3.2 关键页面设计
*   **驾驶舱**：采用“本草纲目”式布局，核心指标置顶，产业分布以环形图呈现，视觉重心集中。
*   **分析页**：采用“左右分屏”设计。左侧为原始文档，右侧为 AI 深度研判，方便用户进行数据溯源与对比。
*   **列表页**：采用“卡片流”布局，通过 Fit Score 标签快速区分项目优先级。

## 4. 如何获取设计稿与源码
由于本系统是基于代码生成的动态 UI，您可以：
1.  **导出源码**：点击 AI Studio 右上角的“设置”菜单，选择“导出到 GitHub”或“下载 ZIP”，即可获得包含所有 UI 实现逻辑的完整 TypeScript/React 源代码。
2.  **查看样式**：UI 设计逻辑主要定义在 `/src/App.tsx` 的 Tailwind CSS 类名中，以及 `/src/index.css` 的全局样式定义中。
